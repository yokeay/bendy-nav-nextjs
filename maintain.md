# Maintain

## v2.4.4

- 日期：2026-04-04
- 迭代类型：小版本
- 目标：开始把旧首页从 `public/dist` 编译产物迁移到可维护的 Next.js + TypeScript 页面与组件。

### 本次修改

- 新增 `src/server/home` 首页数据服务，语义化读取站点设置、默认首页配置、用户首页配置与快捷栏数据。
- 新增 `src/features/home/home-page.tsx` 与 CSS Module，使用 TypeScript 组件重建首页首屏。
- 新增 `app/page.tsx`，让根路径直接由 Next.js 页面接管，不再返回旧 `dist/index.html`。
- 保留 `app/[...path]/route.ts` 与 legacy handler，其余接口和兼容路由暂不动，降低迁移风险。
- 新增 `plan.md` 与本维护文档，开始按迭代记录推进迁移。
- 新增首页客户端请求层与 toast 机制，后续迁移交互时不再直接依赖旧前端状态机。
- 新增首页登录/注册/重置密码弹层、用户菜单与退出登录流程，直接复用现有 legacy API。
- 新增首页全局通知展示能力，并把通知内容改为服务端读配置后直接注入首页数据。
- 移除 `next.config.mjs` 中与当前启动方式冲突的 `standalone` 输出配置，消除生产启动警告。

### 迁移结果

- 首页首屏已经具备独立的 TypeScript 源码入口。
- 页面保留原有视觉方向：背景图、时间区、搜索栏、导航网格、底部 Dock 与备案信息。
- 旧版完整能力仍可通过兼容入口访问，避免首批迁移阻断现有使用路径。
- 新版首页已经具备基本账户入口，可以直接进行登录、注册、密码重置与退出。
- 生产构建与常规 `next start` 启动方式已经重新对齐。

### 风险与后续

- 目前尚未迁移登录弹层、设置中心、拖拽排序、插件容器等高交互能力。
- 仍有大量首页业务逻辑留在 `src/server/legacy/handler.ts`，后续需要继续拆分。
