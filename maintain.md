# Maintain

## v2.4.11

- 日期：2026-04-05
- 迭代类型：小版本
- 目标：先把首页卡片体系从“散落条件判断”收敛成可演进模型，再继续做 1:1 UI 校准。

### 本次修改

- 新增 `docs/home-card-architecture.md`，系统梳理旧版首页的卡片分类、展示态/编辑态/创建态流程，以及新版应采用的卡片领域模型。
- 新增 `src/features/home/home-card-model.ts`，把普通链接卡、应用卡、动作卡、文件夹卡、分组卡的判定和基础能力抽成独立模型层。
- 首页开始接入卡片模型工具，收敛文件夹、动作卡、可编辑能力、可见卡片集合和排序相关判断。
- 按旧版 CSS 的盒模型关系回调新版卡片骨架：卡片容器撑满栅格轨道，但内部图片不再直接拉满卡面高度，减轻标签卡整体偏高和行距失真问题。

### 迁移结果

- 首页卡片不再只是 UI 组件集合，已经开始具备“类型 + 能力 + 容器关系”的领域层基础。
- 后续继续补文件夹、窗口容器、插件卡片和更多动作卡时，不需要再把判断散落进首页主文件。
- 当前 1:1 校准有了更稳定的参照系，后续修高度、宽卡、行距和编辑态浮层会更可控。

### 风险与后续

- 当前卡片模型已经落地为工具层，但渲染层还没有完全拆成 `BaseCardFrame / ActionCard / FolderCard / DockCard` 这种更清晰的组件结构。
- 这次先回收了标签卡高度关系，仍需要继续对照旧版截图核对宽卡和文件夹卡的精确占位。

## v2.4.10

- 日期：2026-04-05
- 迭代类型：小版本
- 目标：补首页文件夹交互缺口，并把本地开发验证链路稳定下来。

### 本次修改

- 首页图标现在支持直接拖到文件夹卡片内，访客本地存储与登录态服务端持久化都会同步更新。
- 文件夹内标签新增“移出文件夹”能力，移动到分类时会真实脱离文件夹并落到目标分组根层，而不是只改 `pageGroup`。
- 移动整个文件夹到其他分组时，会同步修正子项的分组归属，避免父子数据漂移。
- `typecheck` 脚本改为先执行 `next typegen` 再跑 `tsc`，避免首次校验因 `.next/types` 尚未生成而失败。
- `next.config.mjs` 补充 `allowedDevOrigins`，对齐 `localhost` / `127.0.0.1` 双地址的本地开发访问。

### 迁移结果

- 新版首页的文件夹不再只是只读预览容器，已经开始具备旧版“收纳和释放标签”的基础维护能力。
- 本地开发的验证链路变得稳定，`build` 与 `typecheck` 都可以直接作为迭代收口步骤使用。

### 风险与后续

- 当前仍未补齐“文件夹内标签直接拖回桌面根层”的拖放路径，暂时通过右键菜单完成移出。
- Dock 拖到主网格后的落点行为仍然偏保守，后续还需要继续对齐旧版的命中区和反馈细节。

## v2.4.9

- 日期：2026-04-05
- 迭代类型：小版本
- 目标：先把新版首页最明显的布局偏差拉回旧版的固定 icon 栅格节奏。

### 本次修改

- 修正首页尺寸映射逻辑，`1x2` 从错误的纵向大卡恢复为旧版的横向宽卡。
- 新版首页主网格改为固定 icon 宽度、固定列距、按视口只计算列数，不再用会把图标拉大的流式列布局。
- 移除主网格中的重复容器，避免布局叠层导致的尺寸异常。
- 保留当前已迁移的编辑模式、Dock 和动作卡片逻辑，同时让它们运行在新的固定栅格上。

### 迁移结果

- 新版首页图标尺寸已经明显收敛到旧版的视觉比例，不再出现整屏巨大图标卡片。
- 主页结构开始回到旧版“固定节奏”的布局逻辑，为后续继续做 1:1 复刻打下基础。

### 风险与后续

- 当前宽卡、文件夹卡和标签文案仍有局部重叠，说明还有细节样式没完全对齐。
- 下一步需要继续压布局细节，并把拖拽命中区和桌面右键菜单一起拉回旧版行为。

## v2.4.8

- 日期：2026-04-05
- 迭代类型：小版本
- 目标：继续补齐旧版首页对底部 Dock 的维护能力。

### 本次修改

- 新版首页增加 `tabbar` 运行态和持久化逻辑，访客走本地存储，登录用户走 `/tabbar/update`。
- 底部 Dock 改为基于当前运行态渲染，不再只展示初始服务端数据。
- 编辑模式下支持将首页标签加入 Dock。
- 编辑模式下支持从 Dock 中移除标签并在 Dock 内拖拽重排。
- 补充拖拽结束后的状态清理，避免拖放中断后页面残留高亮状态。

### 迁移结果

- 新版首页已经开始具备旧版底部 Dock 的基本维护能力。
- 首页主网格与 Dock 两套快捷入口现在都能在新版内直接维护。

### 风险与后续

- 还没有补 Dock 与主网格之间更完整的拖放联动。
- 旧版更细的拖拽反馈和桌面右键菜单仍待迁移。

## v2.4.7

- 日期：2026-04-05
- 迭代类型：小版本
- 目标：继续把旧版首页的桌面编辑能力迁移到新版首页。

### 本次修改

- 扩展 `src/features/home/home-actions.tsx`，统一承载标签编辑器和分组管理器。
- 首页新增编辑模式，进入后可直接编辑、删除当前可见标签。
- 当前分组内的网格项目支持原生拖拽排序，排序后会同步写回服务端或本地存储。
- 侧边栏新增分组管理入口，并支持创建、编辑、删除分组。
- 标签编辑器新增分组归属选择，允许把标签在首页与不同分组之间迁移。

### 迁移结果

- 新版首页已经开始具备旧版“桌面编辑”核心路径，不再只是查看与少量快捷操作。
- 首页当前可见分组中的标签可以直接在新版页面里完成维护。

### 风险与后续

- 当前拖拽排序先覆盖当前分组内主网格，底部 Dock 和更复杂的嵌套场景仍未接管。
- 旧版更完整的桌面右键菜单、分组内细节行为还需要继续补齐。

## v2.4.6

- 日期：2026-04-04
- 迭代类型：小版本
- 目标：恢复旧首页网格中的特殊动作入口，让新版首页继续逼近旧版交互路径。

### 本次修改

- 新增 `src/features/home/home-actions.tsx`，实现新版首页的动作卡片弹层。
- 网格重新展示旧版的 `添加标签`、`壁纸`、`设置` 三个动作入口，不再直接过滤掉 `tab://` 特殊链接。
- `添加标签` 现在可以在新版首页内直接新增导航项，并按登录状态写回服务端或本地浏览器。
- `壁纸` 现在可以在新版首页内切换默认背景、静态背景库图片和自定义链接背景。
- 未登录用户的首页链接数据开始通过本地 `link` 存储保持，和旧版行为更接近。

### 迁移结果

- 新版首页已经不只是“展示旧数据”，而是能直接执行旧首页里的核心入口动作。
- 旧版首页网格里的几个高频动作现在有了对应的 TypeScript 实现。

### 风险与后续

- 当前 `添加标签` 仅覆盖了新版内最基础的新增导航能力，还没有补拖拽、删除、编辑。
- 背景管理目前主要接管用户侧切换逻辑，旧版更完整的壁纸管理能力仍需继续拆。

## v2.4.5

- 日期：2026-04-04
- 迭代类型：小版本
- 目标：继续让新版首页接管旧版首页可见配置能力，减少对 `tab://setting` 的依赖。

### 本次修改

- 新增 `src/features/home/home-settings.tsx`，构建新版首页设置中心。
- 支持布局、搜索、时间、外观四组核心配置项的可视化编辑。
- 首页改为以 `currentConfig` 作为客户端运行态，配置修改可以即时反映到页面。
- 未登录用户通过本地 `config` 存储保持配置，已登录用户通过 `/config/update` 写回服务端。
- 工具栏新增设置入口，首页不再只有兼容入口才能修改主要展示配置。

### 迁移结果

- 新版首页已经具备旧版首页最核心的展示配置能力。
- 访客和登录用户都可以在新版页面内直接调整首页主要行为，不必退回旧版设置面板。

### 风险与后续

- 目前设置中心仍未覆盖旧后台里的所有站点级配置项。
- 背景库、更多个性化外观设置、第三方登录细项和桌面编辑能力还需要继续迁移。

## v2.4.4

- 日期：2026-04-04
- 迭代类型：小版本
- 目标：开始把旧首页从 `public/dist` 编译产物迁移到可维护的 Next.js + TypeScript 页面与组件。

### 本次修改

- 新增 `src/server/home` 首页数据服务，语义化读取站点设置、默认首页配置、用户首页配置与快捷栏数据。
- 新增 `src/features/home/home-page.tsx` 与 CSS Module，使用 TypeScript 组件重建首页首屏。
- 新增 `app/page.tsx`，让根路径直接由 Next.js 页面接管，不再返回旧 `dist/index.html`。
- 保留 `app/[...path]/route.ts` 与 legacy handler，其余接口和兼容路由暂不动，降低迁移风险。
- 新增 `plan.md` 与本维护文档，开始按迭代记录推进迁移。
- 新增首页客户端请求层与 toast 机制，后续迁移交互时不再直接依赖旧前端状态机。
- 新增首页登录/注册/重置密码弹层、用户菜单与退出登录流程，直接复用现有 legacy API。
- 新增首页全局通知展示能力，并把通知内容改为服务端读配置后直接注入首页数据。
- 移除 `next.config.mjs` 中与当前启动方式冲突的 `standalone` 输出配置，消除生产启动警告。

### 迁移结果

- 首页首屏已经具备独立的 TypeScript 源码入口。
- 页面保留原有视觉方向：背景图、时间区、搜索栏、导航网格、底部 Dock 与备案信息。
- 旧版完整能力仍可通过兼容入口访问，避免首批迁移阻断现有使用路径。
- 新版首页已经具备基本账户入口，可以直接进行登录、注册、密码重置与退出。
- 生产构建与常规 `next start` 启动方式已经重新对齐。

### 风险与后续

- 目前尚未迁移登录弹层、设置中心、拖拽排序、插件容器等高交互能力。
- 仍有大量首页业务逻辑留在 `src/server/legacy/handler.ts`，后续需要继续拆分。
